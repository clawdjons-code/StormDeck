Product thesis

Operational clarity over weather theater.

PAR turns radar from a slow rotating dish into a fast atmospheric sensor. A game engine should turn rapid volumes into an interactive storm world that is time-scrubbable, geospatially aligned, object-aware, and honest about uncertainty.

Answer faster

What changed, where, why it matters, who is affected, what next?

Never hide provenance

Every view shows timestamp, source, field, units, scan age, confidence, and uncertainty.

Bryan’s direction

MWE first. Real data later. Live PAR last.

Direct guidance: build an app using a game engine to render a 3D display of simulated reflectivity data, hard-coding as many parameters as necessary.

Recommended stack

Unreal front end, Python data generator, engine-neutral contract.

Unreal Engine 5 preferred for high-end workstation rendering, cockpit UI, and future VR/video wall. Unity remains acceptable if team velocity wins.

Generator

Python + NumPy produce synthetic fields, metadata JSON, and GPU-friendly payloads.
